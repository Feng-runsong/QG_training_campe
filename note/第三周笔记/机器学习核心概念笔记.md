# 机器学习核心概念笔记

## 1. 什么是机器学习？

- **定义**：让计算机从数据中自动学习规律和模式，并利用这些规律对未知数据进行预测或决策，无需为每个任务显式编写规则。
- **与传统编程的区别**：
  - 传统：数据 + 规则 → 结果
  - 机器学习：数据 + 结果 → 规则（模型）
- **一般流程**：
  1. 数据收集与预处理
  2. 特征工程
  3. 划分数据集（训练/验证/测试）
  4. 选择模型
  5. 训练模型
  6. 评估与调优
  7. 最终测试
  8. 部署与监控

---

## 2. 三大学习范式

### 2.1 监督学习（Supervised Learning）

- **核心**：训练数据包含输入和输出标签，学习映射 \( f: X \to Y \)。
- **任务类型**：
  - 分类（离散输出，如垃圾邮件识别）
  - 回归（连续输出，如房价预测）
- **常用算法**：线性回归、逻辑回归、决策树、随机森林、SVM、KNN、神经网络等。
- **优点**：效果通常较好，部分模型可解释性强。
- **缺点**：标签获取成本高。

### 2.2 无监督学习（Unsupervised Learning）

- **核心**：数据无标签，模型自主发现隐藏结构。
- **任务类型**：
  - 聚类（K-Means、DBSCAN）
  - 降维（PCA、t-SNE）
  - 关联规则挖掘（Apriori）
  - 异常检测（Isolation Forest）
- **应用**：客户分群、图像压缩、推荐系统。
- **评估难点**：缺乏真实标签，常用轮廓系数等。

### 2.3 强化学习（Reinforcement Learning）

- **核心**：智能体通过与环境交互获得奖励/惩罚，学习最大化累积奖励的策略。
- **要素**：智能体、环境、状态、动作、奖励、策略、价值函数。
- **常见算法**：
  - 基于价值：Q-Learning、DQN
  - 基于策略：Policy Gradient、PPO
  - 演员-评论家：A2C、A3C、SAC
- **应用**：游戏AI、机器人控制、自动驾驶。

### 对比表格

| 维度       | 监督学习         | 无监督学习 | 强化学习          |
| ---------- | ---------------- | ---------- | ----------------- |
| 反馈类型   | 正确标签（立即） | 无反馈     | 奖励/惩罚（延迟） |
| 数据独立性 | 独立同分布       | 独立同分布 | 序列相关          |
| 主要挑战   | 过拟合、标签噪声 | 评估困难   | 探索-利用困境     |

## 3. 数据集划分：训练集、验证集、测试集

| 数据集 | 用途                         | 参与参数更新？ | 用于调超参数？ |
| ------ | ---------------------------- | -------------- | -------------- |
| 训练集 | 学习模型参数（权重）         | ✅ 是           | ❌ 否           |
| 验证集 | 评估超参数、选择模型         | ❌ 否           | ✅ 是           |
| 测试集 | 最终评估泛化能力（仅用一次） | ❌ 否           | ❌ 否           |

- **常见比例**：
  - 小数据：60% 训练 / 20% 验证 / 20% 测试
  - 大数据：98% 训练 / 1% 验证 / 1% 测试
- **交叉验证**（数据不足时）：K折交叉验证、留一法。
- **常见错误**：在测试集上调参、划分前未打乱数据、验证集与测试集分布不一致。

## 4. 拟合与泛化

### 4.1 拟合状态

| 状态     | 表现                     | 原因                           | 解决方法                         |
| -------- | ------------------------ | ------------------------------ | -------------------------------- |
| 欠拟合   | 训练误差高，验证误差也高 | 模型太简单、特征不足           | 增加复杂度、更多特征、减少正则化 |
| 过拟合   | 训练误差极低，验证误差高 | 模型太复杂、数据量少、记忆噪声 | 增加数据、正则化、早停、简化模型 |
| 适度拟合 | 两者都低且差距小         | 偏差-方差平衡                  | —                                |

### 4.2 偏差-方差权衡

- **偏差**：预测值与真实值的差距（欠拟合 → 高偏差）
- **方差**：不同训练集上预测的波动（过拟合 → 高方差）
- **总误差 = 偏差² + 方差 + 不可约误差**

### 4.3 泛化

- **定义**：模型在新数据（测试集）上的表现能力。
- **泛化误差 ≈ 训练误差 + 泛化差距**。
- **提高泛化方法**：正则化、早停、交叉验证、集成学习、数据增强。

## 5. 模型评估指标

### 5.1 分类任务（以二分类为例）

混淆矩阵：
- TP：真阳性(正确分类为真)
- TN：真阴性(正确分类为假)
- FP：假阳性(错误分类为真)
- FN：假阴性(错误分类为假)

| 指标                      | 公式           | 适用场景                                   |
| ------------------------- | -------------- | ------------------------------------------ |
| 准确率                    | (TP+TN)/(所有) | 类别平衡时                                 |
| (Precision)精确率(查准率) | TP/(TP+FP)     | (错查)关注预测为正例的准确性（如垃圾邮件） |
| (Recall)召回率(查全率)    | TP/(TP+FN)     | (漏查)关注漏掉正例的代价（如癌症筛查）     |
| F1分数                    | 2×P×R/(P+R)    | 不平衡类别，兼顾精确率和召回率             |

1. `召回率`侧重于,实际有多少正例,我又将其中的多少预测到了.
2. `精确度`侧重与,我预测的正例中,我到底误判了多少.
   因此,在一定程度上,召回率一高,判正的几率就会变高,从而会产生更多的误判,因此精确度会下降
   当我们过分在意分辨率的时候,往往又会错判,导致原本为正的没有被判到,从而使得召回率下降,因此出现一定程度上的互斥性质

**多分类**：宏平均、微平均、加权平均。

### 5.2 回归任务

| 指标 | 公式                                                      | 特点                      |
| ---- | --------------------------------------------------------- | ------------------------- |
| MSE  | $(\frac{1}{n}\sum (y_i-\hat{y}_i)^2)$                     | 对大误差惩罚重            |
| RMSE | $(\sqrt{MSE})$                                            | 与原始单位相同            |
| MAE  | $(\frac{1}{n}\sum |y_i-\hat{y}_i|)$                       | 对异常值鲁棒              |
| R²   | $(1-\frac{\sum (y_i-\hat{y}_i)^2}{\sum (y_i-\bar{y})^2})$ | 解释方差比例，越接近1越好 |
| MAPE | $(\frac{1}{n}\sum\frac{y_i-\hat{y}_i}{y_i})$              | 相对误差$（y_i  ≠ 0）$    |

### 5.3 聚类任务（无监督）

- **轮廓系数**：范围 [-1,1]，越高表示聚类越好
- **Calinski-Harabasz指数**：越大越好
- **Davies-Bouldin指数**：越小越好

### 5.4 排序与推荐

- 精确率@K、召回率@K
- 平均精度均值（mAP）
- 归一化折损累计增益（NDCG）

## 6. 总结速查表

| 概念       | 一句话总结                                |
| ---------- | ----------------------------------------- |
| 机器学习   | 从数据中学习规律，用于预测或决策          |
| 监督学习   | 有标签，学习输入到输出的映射              |
| 无监督学习 | 无标签，发现数据内在结构                  |
| 强化学习   | 通过试错与环境互动，最大化累积奖励        |
| 训练集     | 学习参数的数据                            |
| 验证集     | 调超参数、选择模型的数据                  |
| 测试集     | 最终评估泛化能力的数据（仅用一次）        |
| 欠拟合     | 模型太简单，训练误差就高                  |
| 过拟合     | 模型太复杂，只记住训练集，泛化差          |
| 泛化       | 模型在新数据上的表现能力                  |
| 评估指标   | 量化模型好坏的度量（如准确率、MSE、F1等） |
