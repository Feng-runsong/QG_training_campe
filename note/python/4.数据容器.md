列表(list):

- 列表的定义方式及特点

1. 列表名=[元素1，元素2，元素3]	
2. 可以存放不同类型元素，可重复，有序，元素可以修改

- 列表的索引

1. 正向索引：从e开始
2. 反向索引：从-1开始	

- 列表元素的查看，修改，删除

1. 查看：list1[0]
2. 修改：list1[0]="A"删除：del list1[3]
3. 注意：如果指定的索引值超出范围，将会报错

- 列表的切片

1. 语法：s[start​\:end\:step]

2. 特点：

   tart：开始索引，不指定默认为0(第一个元素的索引)	

   end：结束索引，不指定默认为列表长度(直到列表末尾) 

   step：步长，不指定默认为1

- 列表推导式

  含义：列表推导式就是按照一定规则快速生成一个列表的方法

  格式1：列表名称=[要插入列表的数据 for iin 列表]	

  格式2：列表名称=[要插入列表的数据 for iin 列表if条件]

字符串(str):

- 字符串的特点:
  不可变性(无法修改)，有序性，可迭代性
- 字符串的索引
  正向索引：从开始
  反向索引：从-1开始
- 字符串的切片	
  语法：s[start​\:end\:step]
- 特点：
  start：开始索引，不指定默认为0(第一个元素的索引) end：结束索引，不指定默认为-1(最后一个元素的索引). step：步长，不指定默认为1(-1表示从后向前)

元组(tuple):

- 元组特点:

1. 元素可以重复
2. 有序
3. 不可修改(只读)

- 元组的两个查询方法?	

1. count()：统计指定元素的数量	
2. index()：获取指定元素的索引，如果该元素不存在将报错

- 元组使用注意事项:
  定义单元素元组时，需要在结尾加上逗号，比如('A',)

- 组包与解包:

1. 解包(unpacking)：将容器中的元素分别赋值给多个变量。
2. 组包(Packing)：将多个值合并到一个容器(元组，列表)中。
   在元组解包时，*表示收集剩余的所有元素，允许我们处理不确定数量的元素

集合(set):

- 集合特点:
  无序，不可重复，可修改(不要依赖显示顺序)

- 集合的定义
  定义1：集合名=set()
  注意：空集合的定义不可以使用{},{}表示的是空字典；由于集合是无序的，因此是不支持下标索引访问的。

  定义2：集合名={元素1，元素2，元素3...}

- 集合set操作的运算符&|-

1. &：交集
2. |：并集
3. -：差集	

- 集合推导式的写法:		

1. 变量名称={i表达式 for iin 列表]
2. 变量名称={i表达式foriin列表if条件}

- 集合的常用操作：

添加：s1.add(..)

删除：s1.remove(..)   或   s1.pop()   或   s1.clear()

交，井，差集：s1.intersection(s2)   或   s1.union(s2)   或   s1.difference(s2)[保留s1部分]

字典(dict):

- 字典的存储形式及特点?

1. 字典(dict)存储的是键值对(key:value)形式的数据
2. 特点：键值对，key是唯一的，可修改

- 字典定义的语法:
  定义：dictl={"王林":675,"李慕婉":608,"许立国":478,}
  空集合:dict1={}或dict2=dict()	
  获取：v=dict1["王林"]
- 字典的注意事项

1. value可以是任意类型，而key必须是不可变类型(不能为list,set,dict)
2. 字典内的key不允许重复，如果重复定义，后面的覆盖前面的
3. 字典是没有索引下标的，不能根据索引获取值，只可以根据key获取value

- 字典的常用操作:

1. 添加：字典[key]=value	(key不存在，就会执行新增)	
2. 删除：del 字典[key] / value = 字典.pop(key)。
3. 修改：字典[key]=value    (key存在，就会执行修改)
4. 查询：字典[key] / 字典.get(key)；字典.keys() / 字典.values() / 字典.items()	

- 字典的遍历:
  字典支持for循环遍历
  通过keys() , values() , items() 进行循环遍历
