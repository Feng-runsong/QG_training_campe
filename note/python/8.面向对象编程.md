面向过程编程:

- 核心思想：把一个需求分解成一系列要执行的步骤，然后按照步骤依次执行这些任务(关注的是流程，步骤).
- 适用场景：面向过程编程非常直接，适合简单，线性的任务。

面向对象编程:

- 对象可以理解为现实中具体的人/物在程序中的数字化身(万物皆对象).
- 它把一个人/物的特征(属性)和功能(方法)打包到一起，是面向对象编程的基本单元。(关注的是谁来帮我做这件事儿)

类与对象

- 类：描述的是一组具有相同属性(特征)和方法(功能/行为)的模板。
- 对象：对象是类的实例，是基于类创建出来的(实例对象).


提示：对象是由类创建出来的，创建对象的过程，也称为对象的实例化。一个类可以创建无数个对象。

Python面向对象的四大特性

- 封装(Encapsulation):信息隐藏与访问控制

  核心思想：将数据(属性)和操作(方法)捆绑在一起，并对外部隐藏内部实现细节。

  提高安全性：防止外部代码直接修改对象内部状态。降低耦合：减少不同模块间的依赖关系。

  增强模块性：使代码更易于理解和维护。

  实现：通过_init_方法初始化属性，通常使用下划线(如_attribute或_attribute)表示私有属性，提供公共的 get 和 set 方法来访问和修改。

  ```python
  class Person:
  	def_init_(self,name): 
      	self._name =name
  	def get_name(self):
      	return self._name
  	def set_name(self, new_name):
  		self._name = new_name
  ```

---

- 继承(Inheritance):代码复用与层级关系	

  核心思想：允许一个类(子类/派生类)继承另一个类(父类/基类)的属性和方法。

  代码复用：子类可以直接使用父类已有的功能，无需重复编写。

  建立类之间的层级关系：形成"is-a"关系(例如，"狗是一种动物").

  扩展性：子类可以在继承的基础上添加新的属性和方法，或修改父类的行为。

  实现：使用 class Subclass(Superclass)：语法定义子类。

  ```python
  class Animal
  	def __init__(self,name):
  		self._name =name
  	def sound(self): 
          pass
      
  class Dog(Animal):
      def __init__(self.name,breed):
  		super().__init__(name)
      	self.breed=breed
      
      def sound(self):
          retun  "Woofi"	
  ```

- 多态(Polymorphism):同一接口，不同实现

​	核心思想：同一个操作作用于不同的对象，可以有不同的解释和执行结果。

​	灵活性：可以针对不同类型的对象使用相同的代码逻辑。

​	可扩展性：容易添加新的，具有相同接口但行为不同的类。

​	解耦：调用者无需关心具体对象类型，只需调用统一的接口。

​	方法重写(Method Overriding)
​	子类重新定义从父类继承来的同名方法，提供特定于子类的实现。

​	鸭子类型(Duck Typing)
​	Python特有的特性，只要对象有相应的方法，就能被当作某种类型使用，无需显式的继承关系。

```python
def make_sound(animal):
    return animal.sound() 

class Cat:
	def sound(self):
		return "Meow!" 

class Bird:
	def sound(self):
		return "Tweet!"	
```

- 抽象(Abstraction):关注本质，忽略细节

  核心思想：捕捉事物的关键特征，忽略非本质的细节。它是一种思考方式，也是设计模式。

  简化复杂性：将复杂的系统分解为更小，更易于管理的部分。

  定义清晰的接口：明确一个类应该做什么，而不是如何做。促进模块化设计：使系统各部分职责分明。

  抽象类(Abstract Class)
  包含一个或多个抽象方法(没有具体实现的方法)，子类必须实现这些抽象方法。
  	Shape
  	get_area0->float
  	get_perimeter->float

  在某些语言中(如Java)，接口是纯种的抽象定义，只有方法签名，没有实现。Python中常用抽象基类abc模块来模拟接口。

四大特性协同工作

1. 封装:提供了基础的保护和组织结构，将数据和操作捆绑在一起，隐藏内部实现细节。
2. 继承:实现了代码的复用和层次化，允许子类继承父类的属性和方法，建立"is-a"关系。	
3. 多态:带来了灵活性和可扩展性，同一个操作可以有不同的解释和执行结果，支持方法重写和鸭子类型。	
4. 抽象:指导了设计思路，明确了关注点，捕捉关键特征，忽略非本质细节，简化复杂系统。	

这四大特性共同构成了面向对象编程的基石，使得Python能够构建出结构清晰，易于维护和扩展的大型软件系统。

异常(bug)

- 为什么要捕获异常:
  当程序运行出现异常，提供预案，处理异常，而不是让其中止程序运行

- 如何捕获异常，具体的语法:

  ```python
  try:
  	print("ABC".hello) 
  except NameError as e:
  	print("名称不存在，请检查，具体信息：",e)
  except ZeroDivisionError as e:
  	print("0不能做被除数，请检查，具体信息：",e) 
  except IndexError as e:
  	print("索引错误，请检查，具体信息：",e)
  except Exception as e:
  	print("其他错误，请检查，具体信息：",e) 
  finally:
  	print("无论正常执行还是出现异常，都要释放资源(~")
  ```
